API Reference

Welcome to the official API documentation for api21. This project serves as a highly modular, lightning-fast JSON API backend built on the Go stack.

Architecture Summary

We migrated our architecture from a hybrid Bun + Vite SSR project into a Go standalone engine optimized for low latency and high concurrency:

  • Engine: Go v1.22+ utilizing the Gin Web Framework.
  • Databases: PostgreSQL for persistent data relational storage with GORM ORM.
  • Caching & Rate Limiting: Redis client connection managing distributed sliding window state.
  • Background Workers: Multi-threaded goroutine workers checking server and dependency health autonomously.

Rate Limiting

All endpoints grouped under the /api/ path are governed by a sliding window rate limiter backed by Redis. This protects the service from abuse and ensures reliability.

Header Description
X-RateLimit-Limit Maximum allowed requests per client within the 15-minute window (Default: 200).
X-RateLimit-Remaining The number of remaining requests allowed for your IP within the current window.

If you exceed the limit, the API returns HTTP Status Code 429 Too Many Requests with a JSON response body payload:

{
  "error": "Too many requests, please slow down."
}

Health Check

GET
/api/v1/health

Returns connection diagnostics for database dependencies (PostgreSQL and Redis). Useful for Kubernetes readiness probes or status dashboards.

Response Format

{
  "success": true,
  "data": {
    "status": "ok",
    "postgres": "up",
    "redis": "up"
  }
}

Get Users

GET
/api/v1/users

Retrieves an array containing all registered users in the database.

Query Parameters

No query parameters are currently required.

Create User

POST
/api/v1/users

Creates a new user record. The request payload must be valid JSON.

Request Body Parameters

Parameter Type Required Description
name string Yes Display name of the user. Maximum 255 characters.
email string Yes Unique valid email address. Used for database unique validation index constraints.

Delete User

DELETE
/api/v1/users/:id

Permanently deletes a user from the system by their primary key ID.

Path Parameters

Parameter Type Required Description
id integer Yes The unique numerical primary key of the target user.

	"bytes"
	"fmt"
	"io"
	"net/http"

func main() {
	payload := []byte(`{"name": "Alice Smith", "email": "alice@example.com"}`)
	resp, err := http.Post(
		"http://localhost:3000/api/v1/users",

	fmt.Println(string(body))
}
